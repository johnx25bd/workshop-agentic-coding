# Agentic Coding

**Founders and Coders Open Workshop**

## Overview

This workshop will give you a flavour of our new AI engineering programme and covers agentic coding—the architecture and core concepts behind how coding agents work.

## Background

Coding agents are everywhere. Claude Code, OpenCode, Codex, Cursor, Cline, Aider—new tools keep appearing, each promising to change how software gets built. If you have used one, you have probably had the experience of it working brilliantly on one task and baffling you on the next, with no clear understanding of why.

Most developers are using these tools without a mental model of what is actually happening underneath. What is an agent loop? What is a harness? What happens when the context window fills up? Why does the same prompt produce different results on different runs? What are hooks, skills, and sub-agents, and when do they matter?

These are not questions about any one tool. They are architectural concepts that apply across every coding agent. Once you understand them, you can pick up any agent—current or future—and know what you are looking at. You can configure it deliberately, reason about its failures, and make informed decisions about what to trust it with.

This workshop builds up a complete mental model of how coding agents work, from the simplest possible starting point to the full complexity of multi-agent orchestration. At each layer, we will work with real agents so that every concept is grounded in something concrete.

## Who is this for:

This workshop is for software engineers at any level of experience with coding agents. Whether you have never used one or you are already deep in your workflow with Claude Code or Cursor, the focus is on building a mental model of the underlying architecture — there is something to gain regardless of where you are starting from.

## What we will cover

1. Opening talk: the anatomy of a coding agent and why the architecture matters more than the tool
2. Build up the mental model layer by layer—the agent loop, the harness, context management, configuration, and beyond
3. Configure and use a coding agent on a real task, applying what you have learned
4. Explore advanced patterns: hooks, sub-agents, orchestration
5. Share & discuss
6. AI engineering Q&A
